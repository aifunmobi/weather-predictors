05Using Forecast Revisions

Three concrete modes, in increasing order of confidence required.

A

Programmable predictor

Convert any forecast revision into a +1 / 0 / −1 position and backtest with the shared harness. Most will be noise; some real signals exist.

B

Sizing / timing filter

Use predictor output as a multiplier on an existing fundamental position. Lets you scale exposure 0.5×–1.5× by signal strength.

C

Event-risk timing

Flatten or hedge before known NOAA outlook releases. Nature Comms (2024) documents $12M/year of options premium around these dates alone.

Test before invert

Two of the academic claims in this work fired in the opposite direction. Never invert based on a hunch — run the literal academic strategy first, then check whether the inversion is robust to nearby parameter values. If a lag-sensitivity sweep keeps showing reversal, it's more likely real than a single-lucky-lag artifact.

Add a new predictor

from lib.data import load_oni, monthly_returns
from lib.backtest import run_backtest
import pandas as pd

def build_my_signal():
    oni = load_oni()
    sig = pd.Series(0.0, index=oni.index)
    sig[oni["anom"] >= 1.0] = +1    # strong El Niño → long
    sig[oni["anom"] <= -1.0] = -1   # strong La Niña → short
    return sig

result, equity = run_backtest(
    name="My idea",
    underlying="CC",
    signal=build_my_signal(),
    returns=monthly_returns("CC"),
    freq="monthly",
)
print(result.as_dict())
# → Sharpe, t-stat, hit rate, max DD, verdict
# lib/backtest.py — what run_backtest does

# 1) Align signal with returns on common timeline
# 2) Shift signal forward by 1 period (no lookahead)
# 3) Compute strategy return = position × return
# 4) Annualize: Sharpe × √12 (monthly) or ×1 (annual)
# 5) Report N, t-stat, p-value, hit rate, max DD, Calmar
# 6) Assign verdict:
#      REAL  if |t| ≥ 2 AND |Sharpe| ≥ 0.3
#      WEAK  if 1 ≤ |t| < 2
#      NOISE if |t| < 1

Methodology Caveats

All Sharpe ratios are in-sample, gross of transaction costs, and based on continuous front-month yfinance data (rolls included). Recommended next step: walk-forward validation on 2000–2018 fit window with 2019–2026 holdout.
